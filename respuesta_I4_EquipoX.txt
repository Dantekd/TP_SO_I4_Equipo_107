F- ¿Qué problemas surgieron? 
Al intentar realizar un push se produce el error de "Fetch first" o "non-fast-forward". 
Este ocurre ya que la version de la pagina GitHub con la rama que intentábamos modificar tenia diferente contenido que nuestros respectivos repositorios locales (ya sea por que eran distintas ramas modificadas por distintos integrantes o la misma pero con cambios subidos posterior al pull de la persona subiendo los nuevos cambios). 
Git frena el push para evitar que se sobrescriban de forma no intencionalmente los archivos del repositorio.

G- ¿Qué soluciones pueden aplicar y cual seria la consecuencia de cada solución adoptada? 

La primera solución seria realizar un git pull y resolver los conflictos de manera local antes de intentar hacer el commit y el push de las ediciones locales. 
La segunda manera seria usando el comando git pull --rebase de la rama a modificar para solucionar los errores y evitar sobreescritura 
Por ultimo esta git push --f que es la que terminamos utilizando que sobrescribía cada modificación previa no local por nuestra version del archivo. 
En el ejercicio simultaneo terminamos con la info del GAMMA, el ultimo en ejecutar los comandos y subir su archivo local.
